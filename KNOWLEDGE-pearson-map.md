# Pearson Parameter Map — nabla-type-lite Integration Context

## What the grayscott repo provides

### Asset: `pearson_map.bin`

Generated by `zig build bench-map-pearson` inside the `grayscott` repo.
A pre-generated static binary file to be dropped into `nabla-type-lite/public/`.

**Format:**

```
Bytes 0-23:   Header (little-endian, 6 fields)
              u32 width     (offset 0,  4 bytes)
              u32 height    (offset 4,  4 bytes)
              f32 f_min     (offset 8,  4 bytes)
              f32 f_max     (offset 12, 4 bytes)
              f32 k_min     (offset 16, 4 bytes)
              f32 k_max     (offset 20, 4 bytes)
Bytes 24+:    Pixel data (u8 × width × height, row-major, top-left origin)
              Each byte = V concentration mapped to [0,255]
              0 = no reaction (homogeneous), 255 = maximum reaction activity
```

**Default values:**
- width=1024, height=1024
- f_min=0.01, f_max=0.08 (feed range)
- k_min=0.03, k_max=0.07 (kill range)
- Simulation: 50000 iterations, da=1.0, db=0.5, dt=1.0
- Periodic boundary conditions (torus — no edge artifacts)
- File size: 1,048,600 bytes (24 header + 1024² pixels)
- Gzip'd: ~250 KB

**Regeneration:** Run in grayscott repo:
```bash
zig build bench-map-pearson
# Outputs pearson_map.bin in cwd
```

CLI overrides available:
```bash
zig build bench-map-pearson -- 2048 2048 100000 0.01 0.10 0.03 0.07 custom_name.bin
#                              w    h    iters  f_min f_max k_min k_max output
```

### JS decoder snippet (for the nabla-type-lite component)

```ts
interface PearsonMapHeader {
    width: number
    height: number
    f_min: number
    f_max: number
    k_min: number
    k_max: number
}

async function loadPearsonMap(url: string): Promise<{
    header: PearsonMapHeader
    pixels: Uint8Array
}> {
    const buf = await fetch(url).then(r => r.arrayBuffer())
    const dv = new DataView(buf)
    return {
        header: {
            width:  dv.getUint32(0, true),
            height: dv.getUint32(4, true),
            f_min:  dv.getFloat32(8, true),
            f_max:  dv.getFloat32(12, true),
            k_min:  dv.getFloat32(16, true),
            k_max:  dv.getFloat32(20, true),
        },
        pixels: new Uint8Array(buf, 24),
    }
}
```

## Research background

### What is a Pearson map?

A Gray-Scott reaction-diffusion parameter space visualization. Each pixel represents a different (feed, kill) pair. The X axis maps to kill rate, Y to feed rate. The pixel intensity (V concentration after steady state) reveals what type of pattern emerges at each parameter combination.

The "crescent" region (visible in the generated map) is where interesting patterns form — spots, stripes, mitosis, chaos. Outside the crescent, the system is homogeneous (no pattern). The map lets users pick parameters by clicking on the region that looks interesting.

### Parameter ranges

- Feed (F): 0.01–0.08 (Y axis, top→bottom)
  - Below 0.01: no pattern (system starved)
  - 0.01–0.04: spots, moving spots
  - 0.04–0.06: stripes, mitosis, fingerprint patterns
  - 0.06–0.08: worms, bubbles, chaos
  - Above 0.08: homogeneous (overfed)

- Kill (k): 0.03–0.07 (X axis, left→right)
  - Below 0.03: homogeneous (not enough removal)
  - 0.03–0.05: stable patterns
  - 0.05–0.07: dynamic patterns, replication, chaos
  - Above 0.07: homogeneous (too much removal)

### Why V-channel?

The V concentration correlates with reaction activity: high V = active pattern formation, low V = homogeneous. This gives a clear visual distinction: the "interesting" crescent region appears bright, the homogeneous background is dark. U would be inverted (high U = less reaction).

### Generation parameters

- Grid: 1024×1024 (fine enough for pixel-precise parameter picking)
- Iterations: 50000 (converges in ~25k, 50k is safe margin)
- Diffusion: da=1.0, db=0.5 (matches nabla-type-lite presets)
- Time step: dt=1.0
- Boundaries: periodic (torus — eliminates edge artifacts, every pixel sees same effective neighborhood)
- Convergence: gradient maps converge faster than single-point simulations because neighboring cells stabilize each other through diffusion
