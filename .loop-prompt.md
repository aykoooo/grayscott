## 📐 Key Technical Details

- **Shader is runtime-generated** — inside `generateWgsl()` in `src/gpu/gpu.zig`
- **Current workgroup**: 8×8 (64 threads) — `wg_x`/`wg_y` in GpuState
- **Shared memory tiling IS DONE**: loads (TX+2)×(TY+2) tile into `var<workgroup>` arrays with edge/corner halo
- **Command buffer batching IS DONE**: `gs_gpu_steps(N)` records N dispatches in one command encoder, submits once
- **Ping-pong buffers**: U0→U1 step 1, U1→U0 step 2. Two bind groups (even/odd) swap based on `current_u`
- **Bounds handling**: `select()` for periodic boundaries in both tile loading and computation
- **f16 support**: `has_f16` flag detected via `wgpuAdapterHasFeature()`, but f16 storage was reverted (bandwidth-bound kernel, f16→f32 conversion ALU overhead cancels any savings)
- **Roofline correction**: Kernel is bandwidth-bound (AI=1.56 FLOPs/byte vs ridge at 55.6). Theoretical ceiling = 13.25B cells/sec. Current 2.35B = 17.7% utilization. Room for 5× improvement via temporal blocking + bank conflict fixes.
- **Benchmark hash**: `e16ed0e3c29cc50b5fa2b42791f31ab00b39d488e971b5d3c6017970ed037a43` — sacred, must always match

- [ ] **O.1–O.3** Fix shared memory bank conflicts: pad workgroup arrays to stride-16 to eliminate 2-way bank conflicts on 10×10 tile column access. Expected +10-15%.
- [ ] **P.1–P.3** Temporal blocking (2-step fusion): expand halo to 14×14, compute step t and t+1 before writing global. Expected +50-100%. LBNL/cutile-stencil literature: 1.5-1.8× typical.
- [ ] **Q.1–Q.3** Thread coarsening: each thread computes 2-4 cells. Test horizontal pair (2) and 2×2 block (4). Hou et al. 2017: 1.4-1.8×. Amortizes index calculations.
- [ ] **R.1–R.3** Warp-locality workgroup reshape: sweep 16×4, 32×2 shapes. Adjacent threads in same warp improve coalescing. Expected +10-20%.
- [ ] **S.1–S.3** Subgroup shuffle data sharing: replace SMEM column loads with `subgroupShuffleDown`/`subgroupShuffleUp`. Native: requires naga `enable subgroups;` support (blocker: gfx-rs/wgpu#8202). Browser: Chrome 134+ ships subgroups natively — viable via Emscripten WebGPU target. Expected +10-25%.
- [ ] **T.1–T.3** Register-level output tiling: compute 2×1 or 2×2 cells per thread. Keeps u/v intermediates in registers across adjacent computations. Expected +10-20%.
- [ ] **U.1–U.4** WASM WGSL shader string export: add `src/wasm_shader.zig` exporting `generateWgsl()` variants as WASM functions. Ships optimized shader strings to JS for nabla-type-lite WebGPU pipeline setup. Foundation layer.
- [ ] **V.1–V.4** Full WASM+WebGPU Emscripten pipeline: port GPU pipeline management to Emscripten's browser-webgpu.h. Target `wasm32-emscripten`, verified working with Zig 0.15.2 + Emscripten 4.0.22 (seyhajin/webgpu-wasm-zig reference).
- [ ] **W.1–W.3** CPU SIMD + multithreading WASM fallback: vectorize inner loop with `@Vector(8, f32)`, add wasm-threads support via `SharedArrayBuffer`. Provides 2-4× speedup on non-WebGPU browsers.

### Already Explored & Exhausted (DO NOT RETRY)
- **f16 storage** (Phase K): Correct, zero gain — bandwidth-bound at tested scales.
- **vec2 UV packing** (Phase L): -38% slower, register pressure from swizzling > coalesced f32 reads.
- **Command buffer batching** (Phase F): ALREADY DONE — all dispatches in one submit.
- **Persistent threads**: Device-wide barriers not available in wgpu-native. Not applicable.
- **FP8/INT8 quantization**: Catastrophic cancellation on Laplacian differences. Not applicable.
- **Async copy / pipeline overlap**: WGSL has no async memory primitives. Not applicable.
- **Multi-GPU**: PCIe bottleneck on consumer cards destroys benefit below 4096².
- **Diamond/wave-front tiling**: Designed for high-order (>25pt) 3D stencils. Overkill for 9pt 2D.
- **Tensor core acceleration**: Our FP32 scalar stencil doesn't map to matrix multiply.
- **Semi-stencil algorithm**: Only benefits stencils with radius ≥ 3 (ours is r=1).
- **Device-wide sync (EBISU)**: Requires `syncWorkgroup()` — not in wgpu-native.

### WASM + Browser WebGPU Research Findings
- **Chrome 134+**: `subgroups` feature SHIPPED. `enable subgroups;` works in browser WGSL.
- **Chrome 144+**: Added `subgroup_id` and `num_subgroups` built-in values.
- **Firefox/Safari**: No subgroup support. Need CPU SIMD fallback for these browsers.
- **Emscripten path**: Zig 0.15.2 + Emscripten 4.0+ can compile Zig → WASM that uses browser WebGPU via Emscripten's built-in `webgpu.h`. Reference: seyhajin/webgpu-wasm-zig.
- **wgpu-native explicitly does NOT support emscripten/WASM targets** (gfx-rs/wgpu-native#171). Must use Emscripten's own webgpu.h or go the shader-export route.
