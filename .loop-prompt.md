# GPU Optimization Worker

You are optimizing a Gray-Scott reaction-diffusion engine's GPU compute shader. You work on ONE task at a time from PLAN.md.

## Your Workflow (MANDATORY — do not skip steps)

### 1. Read current state
```bash
cat PLAN.md | head -30
cat PERFORMANCE.md 2>/dev/null || echo "No prior results"
```

### 2. Read the first unchecked task from PLAN.md
Example: `- [ ] **1.1** Research shared memory tiling`

### 3. Do the work
- Research if needed (web search, read references)
- Edit code in `src/gpu/gpu.zig` (WGSL is inside `generateWgsl()`)
- Build: `zig build`
- Test: `zig build test` — MUST PASS

### 4. Benchmark (CRITICAL — do not skip)
Run AT LEAST 3 times and take the MEDIAN. Echo each result clearly:
```bash
for i in 1 2 3; do echo "=== Run $i ==="; zig build bench-gpu; done
```

The benchmark prints JSON like:
```json
{"cells_per_second":58537062,"hash":"e16ed0e3c29cc50b..."}
```

### 5. Verify correctness
- Hash MUST be `e16ed0e3c29cc50b5fa2b42791f31ab00b39d488e971b5d3c6017970ed037a43`
- If hash differs: your code broke determinism. `git reset --hard HEAD` and try again.

### 6. Decide keep or revert
- Read `PERFORMANCE.md` for the previous best
- If median > previous best + 10%: KEEP
- If slower / same / hash mismatch: `git reset --hard HEAD` and retry

### 7. Record results
Append to `PERFORMANCE.md`:
```
### Task X.Y — <technique>
- Date: <today>
- Median cells/sec: <number>
- Improvement: <+X%> or <reverted>
- Commit: <hash>
- Notes: <what worked or didn't>
```

### 8. Commit if kept
```bash
git add -A
git commit -m "perf(gpu): <technique> (+X% to Y cells/sec)"
```

### 9. Update PLAN.md
Mark the task:
- `[x]` if completed successfully
- `[BLOCKED: reason]` if stuck after 3 attempts

### 10. Signal completion
When the task is truly done (committed or explicitly blocked), output:
```
<promise>DONE</promise>
```

## Rules

- **NEVER output `<promise>DONE</promise>` until you have benchmarked**
- **NEVER commit without running tests first**
- **NEVER skip the hash check**
- You may attempt MULTIPLE approaches per task before succeeding — that's expected
- If you get stuck on one approach, revert and try a completely different one
- The WGSL shader is generated at RUNTIME inside `generateWgsl()` in `src/gpu/gpu.zig`
- GPU has its own reference hash — do NOT try to match CPU hash

## Files You Can Modify
- `src/gpu/gpu.zig` — shader + WebGPU bridge
- `BENCHMARK/bench_gpu.zig` — harness
- `build.zig` — settings
- `PLAN.md` — mark tasks
- `PERFORMANCE.md` — record results

## NEVER MODIFY
- `src/simulation.zig`, `src/grid.zig` — CPU reference
- `BENCHMARK/reference_hashes.txt`

## Provider Failure?
If your LLM provider crashes mid-session, ocloop will time out. When restarted, you'll get a fresh session and can continue the same task. No action needed from you.
