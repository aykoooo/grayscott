# GPU Optimization Worker

You are optimizing a Gray-Scott reaction-diffusion engine's GPU compute shader. You work on ONE task at a time from PLAN.md.

## Your Workflow Per Iteration

1. **Read PLAN.md** — find the first unchecked task
2. **Work on that task** — research, edit code, implement
3. **Test** — run `zig build test` (must pass)
4. **Benchmark** — run `zig build bench-gpu` 3×, take median cells/sec
5. **Verify hash** — must match `e16ed0e3c29cc50b...`
6. **Decide**
   - If faster (>10% over previous best): keep changes, commit with conventional commit
   - If slower / hash mismatch / tests fail: revert with `git reset --hard HEAD`, try again or move on
7. **Update PLAN.md** — mark task `[x]` if done, `[BLOCKED: reason]` if stuck
8. **Continue** — ocloop will re-prompt you for the next task

## Files You Can Modify
- `src/gpu/gpu.zig` — WGSL shader (inside `generateWgsl()`), WebGPU bridge
- `build.zig` — compilation settings
- `BENCHMARK/bench_gpu.zig` — benchmark harness
- `PLAN.md` — task tracking (mark tasks done)

## NEVER MODIFY
- `src/simulation.zig`, `src/grid.zig`, `src/map.zig` — CPU reference
- `BENCHMARK/reference_hashes.txt` — sacred reference

## Commit Format
```bash
git add -A
git commit -m "perf(gpu): <what you did>"
```

## Important Notes
- The WGSL shader is generated at RUNTIME inside `generateWgsl()` in `src/gpu/gpu.zig`
- GPU hash does NOT match CPU hash — use ONLY the GPU reference
- Previous best: check PLAN.md or run `zig build bench-gpu` to see current baseline
- You have MANY attempts per task — if one approach fails, revert and try another

When ALL tasks in PLAN.md are complete, add `<plan-complete>All optimization tasks finished</plan-complete>` to PLAN.md.
